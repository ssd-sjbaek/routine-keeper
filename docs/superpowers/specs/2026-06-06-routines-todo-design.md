## 개요

기존 앱에 두 가지 기능 추가:
1. **루틴 편집** — 매일 루틴 항목을 앱에서 직접 추가/삭제
2. **오늘의 할 일** — 그날그날 다른 할 일을 관리하는 체크리스트 (미완료 항목 다음 날 이월)

## 1. 오늘의 할 일

### UI

- 메인 카드에서 루틴 섹션 아래에 위치
- 섹션 제목: "📋 오늘의 할 일" + 오른쪽 "+" 버튼
- "+" 클릭 → 하단에 텍스트 입력 필드 나타남, 엔터 또는 확인 버튼으로 추가
- 각 할 일 항목: 체크박스 + 텍스트 + "✕" 삭제 버튼
- 완료 항목: 줄 그어짐, 당일 계속 표시
- 미완료 항목: 다음 날 이월

### 이월 로직

`renderMain()` 실행 시:
1. `todoList.items` 전체 로드
2. 완료(`done: true`)이고 `doneDate`가 오늘이 아닌 항목 → 목록에서 제거
3. 미완료(`done: false`) 항목 → 그대로 유지 (날짜 무관)
4. 완료이고 `doneDate`가 오늘인 항목 → 줄 그어져서 표시

### Firestore 데이터 구조

```
/users/{uid}/todoList  (단일 문서)
  items: [
    {
      id: string,        // crypto.randomUUID()
      text: string,
      done: boolean,
      doneDate: string   // "YYYY-MM-DD" | null
    }
  ]
```

## 2. 루틴 편집

### UI

- "✅ 매일 루틴" 섹션 제목 오른쪽에 "편집" 버튼
- 편집 버튼 클릭 → 편집 모드 진입:
  - 각 루틴 항목 오른쪽에 "✕" 삭제 버튼 나타남
  - 체크박스 비활성화
  - 목록 하단에 텍스트 입력 필드 + "추가" 버튼
  - 섹션 제목 버튼이 "완료"로 바뀜
- "완료" 클릭 → 편집 모드 종료, 변경사항 Firestore 저장

### 루틴 데이터 형식

기존 `{ name, icon }` 구조에서 단일 문자열로 변경.  
이모지는 이름에 포함해서 자유롭게 입력. (예: `"🏃 달리기"`)

기존 5개 루틴 초기값:
```javascript
["🪟 환기", "🕯️ 향 피우기", "🛏️ 이불 정리", "🧹 청소기 돌리기", "🐱 콩이 화장실 정리"]
```

### Firestore 데이터 구조

```
/users/{uid}/settings  (단일 문서)
  routines: string[]
```

로그인 시 `/users/{uid}/settings` 문서가 없으면 위 초기값으로 자동 생성.
