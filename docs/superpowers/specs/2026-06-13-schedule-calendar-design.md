# 루틴 지킴이 — 스케줄 캘린더 스펙

## 개요

기존 교대근무 기록용 스케줄 화면을 범용 일정 관리 캘린더로 교체한다. 누구나 쓸 수 있는 일반 캘린더 형태로, 일정 등록·수정·삭제를 지원한다.

---

## 1. 화면 구성

### 캘린더 화면 (`screen-schedule`)

| 요소 | 내용 |
|------|------|
| 헤더 | "← 뒤로" 버튼 + "YYYY년 MM월" |
| 월 이동 | 헤더 양쪽 ◀ ▶ 버튼으로 이전/다음 달 이동 |
| 요일 행 | 일 월 화 수 목 금 토 |
| 날짜 셀 | 날짜 숫자 + 일정 칩 (첫 번째 일정 제목, 2개 이상이면 "+N") |
| 오늘 강조 | 오늘 날짜는 `--color-accent` 배경 원형 표시 |

### 날짜 클릭 → 모달 팝업

- 화면 중앙에 모달 오버레이 표시 (배경 반투명 어둡게)
- 모달 헤더: "YYYY년 MM월 DD일 (요일)" + × 닫기 버튼
- 일정 목록:
  - 각 항목: 시간 뱃지 + 제목 + 메모 접힘(탭하면 펼침)
  - 일정 없으면 "등록된 일정이 없어요" 안내 텍스트
- 일정 탭 → 수정/삭제 버튼 인라인 표시
- "+ 일정 추가" 버튼 → 모달 내부에서 등록 폼으로 전환

### 일정 등록/수정 폼 (모달 내 화면 전환)

| 필드 | 타입 | 필수 |
|------|------|------|
| 제목 | 텍스트 입력 | ✅ |
| 시간 | `<input type="time">` | ❌ (비워두면 종일) |
| 메모 | 텍스트 입력 (여러 줄) | ❌ |

- 저장 버튼: 제목 없으면 비활성
- 취소 버튼: 목록 화면으로 돌아감

## 2. 데이터 모델

### Firestore 경로

```
users/{uid}/events/{eventId}
```

### 필드

| 필드 | 타입 | 설명 |
|------|------|------|
| `date` | string | `"YYYY-MM-DD"` |
| `title` | string | 일정 제목 |
| `time` | string \| null | `"HH:MM"` 24시간 형식, 없으면 null |
| `memo` | string \| null | 메모, 없으면 null |
| `createdAt` | timestamp | 생성 시각 |

### 조회 방식

- 월 캘린더 렌더링 시: 해당 월의 모든 이벤트 한 번에 조회
  - `where('date', '>=', '2026-06-01').where('date', '<=', '2026-06-30')`
- 로드한 데이터는 JS 변수에 캐시 (월 이동 시 재조회)

## 3. 기존 코드와의 관계

### 제거

- 기존 교대근무 관련 코드 전체 제거:
  - `saveShift()`, `loadShift()` 함수
  - `renderCalendar()` 내 A조/C조/DO 로직
  - `cal-guide` 안내 텍스트
  - `migrateFromLocalStorage()` 내 schedule 마이그레이션 코드

### 유지

- `screen-schedule` HTML 구조 (내용만 교체)
- `showScreen('schedule')` 호출 시 `renderCalendar()` 실행하는 패턴
- 싸이월드 CSS 변수 (`--color-bg`, `--color-border`, `--color-accent`, `--color-card`) 사용
