# 루틴 지킴이 — 근무 스티커 + 프로필 커스텀 스펙

## 개요

두 가지 독립적인 기능을 추가한다.

1. **근무 스티커**: 캘린더 날짜 셀에서 A/C/D 근무 유형을 탭으로 지정하고, 홈 화면 오늘 날짜 옆에 표시한다.
2. **프로필 커스텀**: 앱 이름("루틴 지킴이")과 닉네임("수조리카의 홈")을 설정 탭에서 변경할 수 있다.

---

## 1. 근무 스티커

### 1-1. 캘린더 UI

- 날짜 셀을 두 영역으로 분리한다.
  - **날짜 숫자 영역**: 탭 시 기존 일정 모달 열림 (변경 없음)
  - **스티커 뱃지 영역**: 탭 시 근무 유형 순환 (없음 → A → C → D → 없음)
- 근무가 설정된 날짜는 셀에 뱃지를 표시한다.
  - A: 붉은 계열 (`#fde8e0`, 텍스트 `#c05030`)
  - C: 초록 계열 (`#e8f0d0`, 텍스트 `#5a7a3a`)
  - D: 보라 계열 (`#ece8f8`, 텍스트 `#5040a0`)
- 근무 미설정 날짜: 셀에 빈 스티커 영역 표시 (탭하면 A부터 시작)
- 월 이동 시 해당 월의 shifts를 events와 함께 새로 조회한다.

### 1-2. 홈 화면

- 날짜 텍스트("2026년 6월 13일 토요일") 바로 옆에 근무 뱃지 표시
- 근무 미설정일에는 뱃지를 숨긴다
- 로그인 직후 오늘 날짜의 shift를 1회 조회
- 탭해도 아무 동작 없음 (표시 전용). 변경은 캘린더에서만 가능

### 1-3. 데이터 모델

**Firestore 경로**

```
users/{uid}/shifts/{YYYY-MM-DD}
```

**필드**

| 필드 | 타입 | 설명 |
|------|------|------|
| `date` | string | `"YYYY-MM-DD"` (events와 동일한 패턴) |
| `shift` | string | `"A"` \| `"C"` \| `"D"` |
| `createdAt` | timestamp | 저장 시각 |

**조회 방식**

- 월 캘린더 렌더링 시: events와 동일한 패턴
  - `where('date', '>=', '2026-06-01').where('date', '<=', '2026-06-30')`
- 홈 화면: 오늘 날짜 1건 조회
  - `where('date', '==', todayKey).limit(1).get()`

**쓰기**

- 순환 탭 시 현재 값 기준으로 다음 값 결정:
  - 없음(문서 없음) → A: `.add({ date: dateKey, shift: 'A', createdAt: serverTimestamp })`
  - A → C: `.update({ shift: 'C' })`
  - C → D: `.update({ shift: 'D' })`
  - D → 없음: `.delete()`

## 2. 프로필 커스텀

### 2-1. 설정 탭 UI

기존 설정 탭에 "프로필" 섹션을 추가한다.

| 항목 | 설명 |
|------|------|
| 앱 이름 | 텍스트 입력, 최대 20자, placeholder "루틴 지킴이" |
| 닉네임 | 텍스트 입력, 최대 20자, placeholder "나의 홈" |

- 각 항목 옆에 "저장" 버튼
- 저장 버튼 탭 → Firestore 업데이트 → 헤더 즉시 반영
- 빈 값으로 저장 시 기본값(`""`)으로 저장, 홈 화면에서 기본 텍스트로 폴백

### 2-2. 홈 화면 반영

- 헤더 타이틀: `appName` 값. 없거나 빈 문자열이면 "루틴 지킴이" 표시
- 헤더 서브타이틀: `nickname` 값. 없거나 빈 문자열이면 "나의 홈" 표시
- 로그인 후 `users/{uid}` 문서 1회 조회하여 반영

### 2-3. 데이터 모델

기존 `users/{uid}` 문서에 필드 추가.

| 필드 | 타입 | 기본값 |
|------|------|--------|
| `appName` | string | `""` (미설정 시 "루틴 지킴이" 표시) |
| `nickname` | string | `""` (미설정 시 "나의 홈" 표시) |
